## 🔍 **특화 서비스 프롬프트 설계 분석**

### **Cluely의 혁신적 설계 원칙:**

#### **1. 극한의 맥락 제한**
```
범용 LLM: "모든 대화 기억하고 활용"
Cluely: "ONLY 최신 맥락" + "CURRENT MOMENT만 우선"
```
→ **이유**: 실시간 회의에서 과거 맥락은 방해가 될 수 있음

#### **2. 계층적 의사결정 시스템**
```
1순위: 최근 질문 답변
2순위: 고유명사 정의  
3순위: 화면 문제 해결
4순위: "모르겠음" 폴백
```
→ **이유**: 회의 중 빠른 결정이 생산성에 직결

#### **3. 극도로 구체적인 제약**
```
- 헤드라인: ≤6단어
- 메인 불릿: ≤15단어  
- 서브 불릿: ≤20단어
- 라인 길이: ≤60자
```
→ **이유**: 회의 중 빠른 스캔 가능성

#### **4. "환각" 방지 시스템**
```
"사실, 기능, 지표를 절대 만들어내지 마세요"
"정보 제한"을 명시적으로 인정
```
→ **이유**: 업무용 도구에서 잘못된 정보는 치명적

### **범용 vs 특화 서비스 프롬프트 비교:**

| 측면 | 범용 LLM | Cluely (특화) |
|------|----------|---------------|
| **목표** | 다양한 대화 | 회의 보조 |
| **응답 형식** | 유연함 | 극도로 구조화 |
| **맥락 활용** | 전체 기록 | 최신만 |
| **불확실성 처리** | 추론 시도 | 즉시 인정 |
| **톤** | 친근함 | 간결함 |
